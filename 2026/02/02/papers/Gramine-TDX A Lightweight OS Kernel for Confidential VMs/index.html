Gramine-TDX: A Lightweight OS Kernel for Confidential VMs

Posted on 2026-02-02 In papers Views: Word count in article: 951 Reading time ≈ 3 mins.

Gramine-TDX: A Lightweight OS Kernel for Confidential VMs

CCS24

Background

Linux 内核庞大且复杂，其中很多功能对于机密计算应用来说都是冗余的，除了增大 TCB 与攻击面之外没有意义。

CVM 攻击面包括：

  • Shared memory
  • PIO MMIO
  • Hypercall
  • Interrupts exceptions
  • CPUID leaves

Heckler(Security’24) 是已经公开的利用 int 0x80 恶意中断进行攻击的例子

Design

Gramine-TDX 是一个轻量级的 LibOS（完全代替 Linux 作为 guest kernel）

Hypervisor 使用 QEMU/KVM

LibOS

  • 不支持多进程 fork 没有 IPC
  • 不能用硬件 例如 GPU
  • 不支持 legacy 16/32bit 只支持 64bit
  • 1:1 mapping
  • 没有 network stack 和 fs stack
  • 不需要修改 APP，不限语言 例如 C, C++, Rust, Python, Java, Go
  • 有且仅有三个 virtio 驱动，并且完全按照 TDX 威胁模型重写，过滤不合法输入

Deployment Model

不需要虚拟机镜像，需要的文件直接从主机文件系统读取

  • 配置信息 （manifest）
  • 共享库（提前准备好的一份 glibc patch 过的）
  • 其他文件 提前加密好 gramine 负责解密

不用虚拟网卡 不用网络配置

Chain of Trust

  • bios -> MRTD
  • gramine kernel 二进制->RTMR1
  • 配置文件 manifest 其中包括了其他普通文件的哈希 RTMR2

td 启动时 tdx module 会验证二进制，记录到这些寄存器中，之后提供给远程验证

对于 RTMR2 td 可以自己 extend 定制化的内容（使用 tdcall 请求 tdx module）

Implementation

manifest 中指定 entry point app hard-code 写死输入 防止 attacker 通过 verbose 等泄漏信息

提供一个 patch 过的 libc 用于适配 gramine 的内核，提前编译成动态库，然后 mount 给虚拟机

virtio-fs

本身用于 host-guest 共享文件 host 是 fuse server，guest 是 fuse client

gramine-tdx 在 host 上用 virtiofsd

文件请求通过 fuse 与 virtio 直接转到 host，因此 guest 不需要 fs stack

intel protected fs

intel 基于自己的硬件已经实现了一套软件的加密 fs 协议 host 上的文件数据是加密的 读取的时候 gramine-tdx kernel 自动透明地会进行解密

  • manifest 中 fs.mount type=”encrypted”
  • manifest 中 sgx.trusted_files 可以记录 file 和对应的 sha256 检察 完整性 （manifest 的完整性也会被检察）

virtio-vsock

本身用于 host-guest 传输网络包

  • guest 上和 host 上都用 socket api 同一套地址 就可以通信
  • host/guest 上都可以用 socat 进行 socket 包和 TCP 包的转化
  1. guest 内 app 全都不需要改 正常通过端口提供服务 例如 8080
  2. gramine-tdx 的 kernel 实现中会进行网络包的转化，并通过 virtio-vsock 转交给 host
  3. host 上 socat TCP4-LISTEN:8080,reuseaddr,fork VSOCK-CONNECT:123:1234 可以在 host 上的 8080 访问 guest 内的服务

sum

无法复制加载中的内容

减小 TCB 减小攻击面 但是 fuse 和 socat 会带来 overhead

Security Analysis

基本就是控制 input、检查 input 合法

Evaluation

Testbed：

  • Intel Xeon Platinum 8570 CPU , 56 cores , 1 TB (16 channels × 64 GB/DIMM) DRAM
  • Host：Ubuntu 24.04, Intel TDX-enabled Linux kernel v6.8.
  • Intel TDX-patched QEMU v8.2.
  • Guest： Ubuntu 24.04,intel TDX-enabled guest Linux v6.8
  • Intel TDX Module v1.5 (build_num 698).
  • vCPU 数量与 Application 需要线程数量相等

Varaints

Microbenchmarks

In-memory 计算开销小于 5%

IO 性能差，在于 gramine-tdx 中的 virtio-fs 驱动实现过于简单

AI Frameworks

  • PyTorch: AlexNet for image classification.
  • OpenVINO,TensorFlow:ResNet-50 for inference
  • TensorFlow:BERT for inference
  • Candle:LLaMA

大部分情况下几乎和 native 一致，平均性能 overhead 在 10%

其他 Application 例如 databse 和 web server，由于需要 socat 的转发，并且 gramine-tdx 的 virito-vsock 设计过于简单，性能会降至 10%

Conclusion

Gramine-TDX 为 TDX 机密虚拟机定制了 guest kernel，大幅减小了攻击面，提升了安全性。

由于重点在安全和代码质量上，性能方面看来目前只适合不需要网络的纯计算任务（例如模型推理）
